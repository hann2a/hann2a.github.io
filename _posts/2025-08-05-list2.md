## 카운팅 정렬 

항목들의 순서를 결정하기 위해 **집합에 각 항목이 몇 개씩 있는지 세는 작업**을 하여, 선형 시간에 정렬하는 효율적인 방식 

- 각 원소의 빈도수를 이용하여 정렬하는 알고리즘 

### 카운팅 정렬 특징 

- 데이터 범위 k가 크지 않은 경우 매우 효율적

- 크기 기준이라 개수만 카운트 해주면 되기 때문에 효율적(데이터에 한 번씩만 접근)

- 시간 복잡도 O(n+k)라 데이터의 개수와 데이터 범위에 따라 결정됨 

- n은 리스트 길이, k는 정수의 최대 값 

- 정렬 과정에서 안정성을 보장할 수 있으며, 주로 정수형 데이터에 사용됨 

**용도**
- 데이터의 값 범위가 제한적일 때 빠른 정렬이 필요한 경우 사용됨

### 카운팅 정렬 제한 사항 

1. 정수나 정수로 표현할 수 있는 자료에 대해서만 적용 가능 
- 각 항목의 발생회수를 기록하기 위해, 정수 항목으로 인덱스 되는 카운트들의 배열을 사용하기 때문이다. 

2. 카운트들을 위한 충분한 공간을 할당하려면 집합 내의 가장 큰 정수를 알아야 함 

### 시간 복잡도 

```latex
O(n+k) # n은 리스트 길이, k는 정수의 최댓값
```

### 카운팅 정렬 과정 예시 
[0, 4, 1, 3, 1, 2, 4, 1]을 카운팅 정렬하는 과정 

1단계: Data에서 각 항목들의 발생 횟수를 세고, 정수 항목들로 직접 인덱스 되는 카운트 배열 COUNTS에 저장한다. 

2단계: 정렬된 집합에서 각 항목의 앞에 위치할 항목의 개수를 반영하기 위해 COUNTS의 원소를 조정한다.

3단계: DATA의 뒤부터 앞까지 돌면서 카운트를 보고 템프를 만든다. 

```python
def counting_sort(DATA, TEMP, k):
  # DATA = []  입력 배열 
  # TEMP = []  정렬된 배열 
  # COUNTS = [] 카운트 배열 
  
  COUNTS = [0] * (k+1) 

  for i in range(len(DATA)): # DATA[i] 발생횟수 기록 
    COUNTS[DATA[i]] += 1
  
  for i in range(1, k+1): # COUNTS 값 조정 (누적)
    COUNTS[i] += COUNTS[i-1]

  for i in range(len(DATA)-1, -1, -1): # 3단계
    COUNTS[DATA[i]] -= 1
    TEMP[COUNTS[DATA[i]]] = DATA[i]
 ```
```python
 def counting_sort(input_arr, k):
    '''
    카운팅 정렬 함수
    input_arr : 정렬할 입력 배열 (각 원소는 0 이상 k 이하의 정수)
    k : 입력 배열 내 원소의 최대값 (데이터 범위를 의미)
    '''
    # 1. k+1 크기의 카운팅 배열을 0으로 초기화 (인덱스 0부터 k까지 사용)
    counting_arr = [0] * (k + 1)

    # 2. 입력 배열 내 각 원소의 빈도수를 counting_arr에 기록
    # (각 요소가 몇 번 나왔는지 확인해야 한다.)
    for num in input_arr:
        counting_arr[num] += 1

    # 3. 누적 합을 계산하여 각 원소가 정렬된 배열 내에서 차지할 위치를 결정
    # (누적 합 배열을 만들어야 한다.)
    for i in range(1, k + 1):
        # 이전까지 나온 수들의 합을 누적하는 코드
        counting_arr[i] += counting_arr[i - 1]

    # 4. 결과 배열 초기화: 입력 배열과 같은 크기로 생성
    result_arr = [0] * len(input_arr)

    # 5. 입력 배열을 역순으로 순회하며, 각 원소를 결과 배열의 올바른 위치에 배치
    # (역순 순회는 정렬의 안정성을 보장하기 위한 기법)
    # for i in range(len(input_arr) - 1, -1, -1):
    for num in reversed(input_arr):
        counting_arr[num] -= 1  # 누적 합에서 해당 원소의 위치 인덱스를 하나 감소시킴
        result_arr[counting_arr[num]] = num

    return result_arr

# 테스트 예시
a = [0, 4, 1, 3, 1, 2, 4, 1]
print('정렬 결과:', counting_sort(a, 5))  # 출력 예시: [0, 1, 1, 1, 2, 3, 4, 4]
```

## Baby-gin Game

0과 9 사이의 숫자 카드에서 임의의 카드 6장을 뽑았을 때, 3장의 카드가 연속적인 번호를 갖는 경우를 run이라고 하고, 3장의 카드가 동일한 번호를 갖는 경우를 triplet이라고 한다. 

그리고 6장의 카드가 run과 triplet로만 구성된 경우를 baby-gin으로 부른다.

6자리 숫자를 입력 받아 baby-gin 여부를 판단하는 프로그램을 작성하라. 

### baby-gin with 순열 
```python
# baby gin with permutation(순열)

N = 3
card = [4, 5, 6]


# baby gin with triplet or run

run = False
tri = False

for i in range(N):
    for j in range(N):
        # if j != i: continue
        if j != i:
            for k in range(N):
                if k != i and k != j:
                    print(card[i], card[j], card[k])

                    # run 검사 
                    if card[i] + 1 == card[j] and card[i] + 2 == card[k]:
                        run = True
                    
                    # triplet 검사 
                    # card[i] == card[j] and card[i] == card[k]
                    if card[i] == card[j] == card[k]:
                        tri = True


                    if run:
                        print('런')
                    
                    if tri:
                        print('트리플')
```

## 탐욕 알고리즘 

여러 경우 중 하나를 결정해야 할 때마다 그 순간에 최적이라고 생각되는 것을 선택해 나가는 방식으로 진행하여 최종적인 해답에 도달하는 방식 

### 탐욕 알고리즘

- 최적해를 구하는 데 사용되는 근시안적인 방법 

- 각 선택의 시점에서 이루어지는 결정은 지역적으로는 최적이지만, 그 선택들을 계속 수집하여 최종적인 해답을 만들었다고 하여, 그것이 최적이라는 보장은 없음 

- 일반적으로, 머릿속에 떠오르는 생각을 검증 없이 바로 구현하면 Greedy 접근

### 탐욕 알고리즘 과정 

1. 해 선택: 현재 상태에서 부분 문제의 최적 해를 구한 뒤, 이를 부분해 집합에 추가한다. 

2. 실행 가능성 검사: 새로운 부분 해 집합이 실행 가능한지를 확인한다. 곧, 문제의 제약 조건을 위반하지 않는지를 검사한다. 

3. 해 검사: 새로운 부분 해 집합이 문제의 해가 되는지를 확인한다. 아직 전체 문제의 해가 완성되지 않는다면 1의 해 선택부터 다시 시작한다. 

### 탐욕 알고리즘 예시 

어떻게 하면 손님에게 거스름돈으로 주는 지폐와 동전의 개수를 최소한으로 줄일 수 있을까? 

1. 해 선택 
    현재 고를 수 있는 가장 단위가 큰 동전을 하나 골라 거스름돈에 추가 

2. 실행 가능성 검사 
    거스름돈이 손님에게 드려야 하는 액수를 초과하는지 확인
    초과한다면 마지막에 추가한 동전을 거스름돈에서 빼고 돌아가서 현재보다 한 단계 작은 단위의 동전을 추가 

3. 해 선택 
    거스름돈을 확인해서 액수에 모자라면 추가할 동전을 고름 

### 탐욕 알고리즘을 활용한 Baby-gin 접근 

- 6개의 숫자는 6자리의 정수값으로 입력됨 

- counts 배열의 각 원소를 체크하여 run과 truiplet 및 baby-gin 여부를 판단한다. 
- 
```
TRI: Counts[i] >= 3
RUN: Counts[i] >= 1 and Counts[i+1] >= 1 and Counts[i+2] >= 1
```

```python
num = 456789 
c = [0] * 12 # 6자리 수로부터 각 자리 수를 추출하여 개수를 누적할 리스트 

# dummy 칸을 만들어 두면 코드가 단순해지는 경우가 있다. 

for i in range(6):
  c[num % 10] += 1 # num % 10: num의 1의 자리를 알아내는 연산 
  num // = 10 # 1의 자리를 없애는 코드 
```

- while을 쓰는 이유 - 같은 자리에서 시작할 수 있어야 해서 

```python
i = 0 
tri = run = 0
while i < 10:
  if c[i] >= 3: # triplet 조사 후 데이터 삭제 
    c[i] -= 3
    tri += 1
    continue
  if c[i] >= 1 and c[i+1] >= 1 and c[i+2] >= 1: # run 조사 후 데이터 삭제 
    c[i] -= 1
    c[i+1] -= 1
    c[i+2] -= 1
    run += 1
    continue
  i += 1

if run + tri == 2: print("Baby Gin")
else: print("Lose")
```

### Baby-gin 문제에서 자주 실수하는 오답 

- 입력받은 숫자를 정렬한 후, 앞뒤 3자리씩 끊어서 run 및 triplet을 확인하는 방법을 고려할 수도 있다. 

[6, 4, 4, 5, 4, 4]
정렬하여 [4, 4, 4, 4, 5, 6]을 얻어내면 쉽게 baby-gin을 확인할 수 있다. 

[1, 2, 3, 1, 2, 3]
정렬하면 [1, 1, 2, 2, 3, 3]로서, 오히려 baby-gin 확인을 실패할 수 있다. 

위의 예처럼, 탐욕 알고리즘적인 접근은 해답을 찾아내지 못하는 경우도 있으니 유의해야 한다. 

# 노션 학습 

### 안정적인 정렬이란?

- 안정적인 정렬은 동일한 키(값)을 가진 원소들이 정렬 후에도 입력 배열 내에서의 상대적 순서를 그대로 유지하는 정렬 

- 예를 들어, 동일한 성적을 가진 학생들이 원래 입력된 순서대로 출력되어야 할 때 안정적인 정렬이 필요. 

### 누적합을 활용하여 배치하는 원리 

- 누적합 배열의 값 `counting_arr[i]`는 0부터 i까지의 값들이 배열에 등장한 **총 개수**

- 누적합 배열은 정렬된 결과에서 각 원소가 최종적으로 들어가야 할 위치를 결정하는 기준
  - 특정 값 v의 누적합이 x라면, v는 정렬된 배열에서 x번째 위치(0부터 시작하는 인덱스로는 x-1)에 마지막으로 배치되어야 한다. 

- 입력 배열은 역순으로 순회시키면서 각 원소 v에 대해 counting_arr[v]의 값을 1 감소시키고, 그 위치에 v를 배치한다. 이렇게 하면 v보다 작은 값들의 개수에 기반해 정확한 위치에 원소가 들어가게 되고, 동일한 값의 원소들 간에도 원래 순서가 되어 안정적인 정렬이 이루어진다. 

## 누적합의 원리와 활용 

### 누적합의 기본 개념 

어떤 배열의 각 인덱스까지의 모든 값의 합을 저장한 배열 

원본 배열이 `[a, b, c, d]`일 때, 누적합 배열은 `[a, a+b, a+b+c, a+b+c+d]`가 된다. 

활용: 누적합은 특정 구간의 합을 빠르게 계산하거나, 데이터의 분포를 누적해서 나타내어 **각 원소가 전체에서 차지하는 위치를 결정**하는데 사용됨 

### 카운팅 정렬에서 누적합의 역할 
카운팅 정렬은 비교 기반 정렬 알고리즘이 아니라, 데이터의 빈도수를 이용해 정렬하는 알고리즘

**빈도수 배열 생성**

- 입력 배열의 각 원소가 몇 번 등장하는지 기록 
  - `[1, 4, 1, 3]`에 대해 빈도수 배열은 
  - `count[1] = 2, count[3] = 1, count[4] =1`로 구성 
  
- 누적 합 배열 구성 
  - 빈도수 배열을 누적합 형태로 업데이트 
  - 누적합 배열의 각 인덱스 i는 '0부터 i까지의 원소들이 입력 배열에 몇 개 있는지'를 나타냄 
  - 빈도수 배열이 `[0, 2, 0, 1, 1]`일 경우, 
  - 누적합 배열은 `[0, 2, 2, 3, 4]`

- 최종 위치 결정 
  - 누적합 배열의 값은 각 원소가 정렬된 배열에서 오른쪽 경계(마지막 인덱스 +1) 역할을 함 
  - 예를 들어, 누적합 배열의 값이 `3`인 경우 
  - 이는 정렬된 배열에서 그 값 **이하**의 원소가 3개 존재함을 의미하며, 
  - 해당 원소는 실제 배치 시 `3-1=2`에 위치하게 됩니다. 

# 오늘의 코드 리뷰 

### counting 정렬 

```python
def counting_sort(input_arr, k):
  count_arr = [0] * (k+1)

  for num in input_arr:
    count_arr[num] += 1
  
  for i in range(1, k+1):
    count_arr[i] += count_arr[i-1]

  result_arr = [0] * len(input_arr)

  for num in reversed(input_arr):
    count_arr[num] -= 1
    result_arr[count_arr[num]] = num 

  return result_arr

arr = [0, 4, 1, 3, 1, 2, 4, 1]
print('정렬 결과:', counting_sort(arr, 5)) # [0, 1, 1, 1, 2, 3, 4, 4]
```

### 구간합 
[4835 구간합](https://swexpertacademy.com/main/learn/course/lectureProblemViewer.do)
```python
T = int(input())

for test_case in range(1, T + 1):
    N, M = map(int, input().split())          # N: 배열 길이, M: 구간 길이
    nums = list(map(int, input().split()))

    # 구간 합(sliding-window)들을 모두 구하는 함수
    def window_sums(arr, window):
        sums = []
        for i in range(0, len(arr) - window + 1):
            curr = sum(arr[i : i + window])   # window 크기만큼 잘라서 합
            sums.append(curr)
        return sums

    sums = window_sums(nums, M)
    answer = max(sums) - min(sums)            # 최대-최소 차이

    print(f"#{test_case} {answer}")
```

```python
충전으로 조금 밖에 못감 (k만큼)
0에서 출발 N까지 이동  
정류장 M개(번호 줌)
최소한 몇 번의 충전을 해야 종점에 갈 수 있는가? 
충전기 설치 잘못돼서 도착 못하면 0 출력 
출발지에는 항상 충전기가 있고 충전횟수 미포함 

K: 충전하고 갈 수 있는 정류장의 수 
N: 종점 
M: 충전기 수 
"""
T = int(input())
for test_case in range(1, 1 + T):
    K, N, M = map(int, input().split())
    bus_stop = list(map(int, input().split()))

    # 구간 간격 검사 하기 
    # 충전소와 충전소, 충전소와 종점까지의 거리 > 가능한 거리? 
    # 시작점 
    prev = 0 
    possible = True
    for stop in bus_stop + [N]:          # 종점 포함
        if stop - prev > K:
            possible = False
            break
        prev = stop

    if not possible:
        print(f"#{test_case} 0")
        continue                         # 여기서 continue는 이 밖의 for문인 
                                         # 다음 test_case로 넘어가게 함 
    # ──────────────────────────────────────────────

    # 충전소 존재 여부 배열
    # 시작점도 세므로 + 1
    stop_arr = [0] * (N + 1)
    for stop in bus_stop:
        stop_arr[stop] = 1

    # 실제 이동 & 충전 횟수 카운트
    now_position = 0
    charge_count = 0
    # else문: 범위 내에 없었으면 charge_count
    while now_position + K < N:              # 종점에 도달할 때까지
        # K 이내에서 가장 멀리 있는 충전소 탐색
        for index in range(now_position + K, now_position, -1):
            if stop_arr[index]:
                charge_count += 1
                now_position = index
                break
        else:                            # for 를 다 돌았는데 충전소 없음
            charge_count = 0
            break

    print(f"#{test_case} {charge_count}")
```
