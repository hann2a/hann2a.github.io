### 리스트

    - 모든 종류의 데이터를 담을 수 있음
    
    중첩 리스트 Nested List
    
    - 다른 리스트를 값으로 가진 리스트
    
    슬라이싱으로 여러 값 한 번에 바꾸기 가능 
    
    - [주소, 주소, 주소]

### 튜플

    - 여러 개의 값을 순서대로 저장하는 변경 불가능한 자료형
    - 모든 종류의 데이터를 담을 수 있음
    - 소괄호 없이도 만들 수 있다
    - 단일 요소 튜플을 만들 때는 후행 쉼표를 써야 한다
    - 불변 자료형인 이유
        - 파이썬 내부 동작과 안전한 데이터 전달에 사용됨
        - 여러 개의 값이 동시에 쓰이는 순간 값을 묶어주어야 함

### Range
    
    연속된 정수 시퀀스를 생성하는, 변경 불가능한 자료형 
    
    `range(start, stop, step)`
    
    - Step이 음수일 때
        - 숫자가 start부터 stop을 향해 감소
        - 이 경우, start 값은 stop 값보다 반드시 커야 함

### dict

    - 순서와 중복이 없는 변경 가능한 자료형
    
    key의 규칙 
    
    - 변경 불가능한 자료형만 사용 가능 str, int, float, tuple
    
    딕셔너리 값 추가 및 변경 
    
    - 언제 딕셔너리를 사용할까요?
        - 데이터에 순서가 필요 없고, 각 데이터에 의미 있는 이름을 붙여 관리하고 싶을 때 사용

### Set
    
    순서와 중복이 없는 변경 가능한 자료형
    
    - 집합 연산할 때 쓰임
    - 중괄호 안에 값들을 쉼표로 구분하여 만듦
    
    집합 연산 
    
### None

    - 내용이 없는 빈 상자

### Boolean
    - 온오프 스위치
    - 프로그램의 흐름을 제어하는 조건문에서 맞다, 틀리다를 판단하는 것

### Collection
    
    여러 개의 값을 하나로 묶어 관리하는 자료형들을 통칭하는 말 
    
    | 컬렉션 명  | 변경 가능 여부  | 순서 존재 여부  |
    | --- | --- | --- |
    | str | x | o |
    | list | o | o |
    | tuple | x | o |
    | dic | o | x |
    | set | o | x |

### 형변환 Type Conversion
    
    한 데이터 타입을 다른 데이터 타입으로 변환하는 과정
    
    **암시적 형변환**
    
    - 파이썬이 연산 중에 자동으로 데이터 타입을 변환하는 것
    - 데이터 손실을 막기 위해 더 정밀한 타입으로 자동 변환해주는 규칙
    - 작은 정수 상자와 큰 실수 상자의 내용물을 합칠 때, 더 안전하게 담을 수 있는 큰 실수 상자로 알아서 옮겨 담는 것과 같다
    
    `print(3 + 0.5) = # 8.0`
    
    `print(True + 3) # 4`
    
    `print(True + False) # 1`
    
    명시적 형변환

### 비교 연산자
    
    `==` 같음 
    
    `! =` 같지 않음
    
    `is` 같음
    
    `is not` 같지 않음 
    
    `==` 연산자 
    
    - 값이 같은 지를 비교(동등성)
    - 논리적으로 같다
    
    `is`  연산자 
    
    - 객체 자체가 같은 지를 비교(식별성)
    - 메모리 주소를 비교
    - 정체성 비교
    - 싱글턴 객체를 비교할 때
        - 프로그램 전체에서 오직 1개만 존재하도록 만들어진 특별한 객체
        - None, True, False
    - b= a처럼 b가 a를 참조할 때는 쓸 수 있음
